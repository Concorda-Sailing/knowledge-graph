What do these terms mean?
Tier A (fan-out ≥ 10)
Hot nodes — heavily depended-on. Editing one ripples through 10+ call sites. Hand-author dossiers; assume any edit here is high-blast-radius.
Tier B (fan-out 3–9)
Mid-fan — common-but-not-critical wrappers, mid-traffic models, schemas. LLM-drafted dossiers acceptable; review-via-graphui ratifies them.
Tier C (fan-out 1–2)
The long tail — one-off endpoints, narrow helpers. Drafted by Haiku or local models in bulk; review-via-graphui as a sweep, not per-node.
Tier *
Logigraph rules and ontology nodes — tier-independent because they describe invariants, not code. Always hand-curated.
Fan-out
Number of distinct callers/dependents the node has, derived from nodes/_index/dependents.json. Drives tier assignment. Higher = bigger blast radius if you change the contract.
Kind
What the node is: model (SQLAlchemy class), service (function/wrapper), endpoint (HTTP route handler), schema (Pydantic class), component (React component), hook (React hook), test (Playwright spec), rule / ontology (logigraph nodes).
Dossier states
current — human-reviewed, hash matches the live code · llm_drafted — written by an LLM agent, not yet reviewed · unreviewed — has a frontmatter but flagged as not-yet-reviewed · stale — the code's structural_hash has shifted since last_reviewed_against_hash was pinned · missing — node exists, no dossier file yet.
