# InviteLandingPage.clickSignUp

## Purpose

The `clickSignUp` method simulates a user clicking the sign-up button on the invite landing page. It is used to transition from the initial invite view to the registration flow. Unlike `accept()`, which handles the confirmation of an existing invite, this method initiates the creation of a new account/identity via the invite link.

## Invariants

- **Triggers a navigation/network event.** The method calls `this.page.waitForLoadState('networkidle')` to ensure the transition to the sign-up flow is complete before returning.
- **Requires a valid invite context.** This method is part of the `InviteLandingPage` POM, which assumes the user is currently viewing a landing page generated by a specific invite token.

## Gotchas

- **Implicit dependency on `networkidle`.** Because the method waits for `networkidle`, any background polling or long-lived connections active during the sign-up transition may cause the test to hang or timeout.

## Cross-cutting concerns

- **Auth**: Initiates the transition from an unauthenticated state to the registration/onboarding flow.
- **Side effects**: Triggers the transition to the sign-up view, which is the first step in the user onboarding funnel.

## External consumers

None known.
