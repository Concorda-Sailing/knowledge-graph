# send_email

## Purpose

The primary service for dispatching emails via three distinct modes: `test` (logging to a file), `console` (printing to stdout), and `sendgrid` (production delivery). It handles the conversion of HTML bodies to plain text via `_html_to_text` to ensure compatibility with email clients. Use this function when a process requires sending a notification that includes attachments or a specific `reply_to` address.

## Invariants

- **Mode-dependent behavior**: The function's side effect is strictly governed by `config["mode"]`.
- **HTML to Text conversion**: The `text_body` is generated by calling `_html_to_text(html_body)` and must be used for the `text/plain` part of the message to ensure accessibility.
- **Attachment structure**: Attachments must be passed as a list of `EmailAttachment` objects, containing `filename`, `mime_type`, `method`, and `content`.
- **Order of content**: In `sendgrid` mode, the `text/plain` content must be added to the message before the `text/html` content to satisfy SendGrid's rendering preference for the alternative part.

## Gotchas

- **Timezone mismatch in bodies**: Per commit `6c314f5`, ensure that any time-based data (like `.ics` files or text-based timestamps) rendered in the `html_body` is converted to the organization's local timezone before calling this function, otherwise, the email will display UTC instead of local time.
- **Test mode file path**: In `test` mode, the function writes to a file path defined by `os.environ.get("TEST_MAIL_LOG", "/data/mail.log")`. If this path is not writable or the environment variable is missing, the test runner may fail.

- **Auth**: None.

- **Side effects**: Triggers email delivery for `event_crew` notifications (per commit `8f84d2d`) and handles `invite` links for boat crew/requests (per commit `8c29970`).

## Open questions
