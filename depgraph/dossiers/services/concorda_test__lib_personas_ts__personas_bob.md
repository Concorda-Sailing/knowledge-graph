# personas.bob

## Purpose

Provides a pre-configured Playwright persona for "Bob," representing a boat owner. It uses the `asPersona` helper to wrap a browser instance with a specific authentication state file (`boat-owner.json`). This allows tests to simulate a logged-in owner without manual login steps during the test execution.

## Invariants

- **Uses `boat-owner.json`** — The persona is strictly tied to the auth state file located in `AUTH_STATES_DIR`.
- **Returns a Playwright `BrowserContext` wrapper** — The result of `personas.bob(browser)` is an object containing the context and a `close()` method.
- **Requires a `Browser` instance** — The function must be passed a Playwright `Browser` object to initialize the context.

## Gotchas

- **Dependency on `asPersona`** — The persona'ality is defined by the underlying `asPersona` logic; any changes to how state files are loaded will affect Bob's identity.
- **Auth state freshness** — Per commit `ab4d7ce`, this helper was added to support two-context tests; if the `boat-owner.json` file is not regenerated by the global setup, the persona will fail to be authenticated.

## Cross-cutting concerns

- **Auth**: Relies on the existence of a valid `boat-owner.json` auth state file.
- **Side effects**: Used to drive E2E specs for features like the "Inbox accept/cancel flows" (see commit `03a3cdd`).
