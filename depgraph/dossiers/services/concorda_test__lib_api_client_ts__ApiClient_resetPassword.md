# ApiClient.resetPassword

## Purpose

The `resetPassword` method completes the authentication recovery flow by submitting a new password and a reset token. It is the second half of the password recovery loop, following `forgotPassword`. Use this method to test E2E flows where a user must recover access to an account, ensuring the new credentials work for subsequent authenticated requests.

## Invariants

- **Method is `POST /api/auth/reset-password`**.
- **Requires both `token` and `password` strings.**
- **Returns `Promise<void>`**. A successful call indicates the password has been updated and the token is now invalidated.
- **Relies on the `reset_token` provided by the API.** The token is the bridge between the `forgotPassword` call and this method.

## Gotchas

- **E2E flows must use `forgotPassword` first.** Per commit `ba50bcc`, the full loop (forgot-password $\rightarrow$ reset-password) is required to validate the UI flow.
- **The API echoes the token for testing.** As noted in the docstring, the test API returns the `reset_token` during the `forgotPassword` call so that E2E tests can immediately proceed to call `resetPassword` without needing to intercept actual emails.

## Cross-cutting concerns

- **Auth**: Requires a valid `reset_token` generated by the server.
- **Side effects**: Successfully calling this changes the user's credentials, which will invalidate any existing sessions/tokens for that user.
